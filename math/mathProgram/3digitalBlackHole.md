# 数字黑洞：三位数为什么总会走向 495

## 一、故事内容

有一位印度数学家叫 **D. R. Kaprekar（卡普雷卡尔）**，他非常喜欢研究“看起来普通、但反复操作后会出现固定结果”的数字游戏。

他本名 **Dattatreya Ramchandra Kaprekar**，生于 **1905 年**，卒于 **1986 年**，主要研究与发表活跃于 **20 世纪中期（约 1940 年代到 1970 年代）**。他长期在中学任教，不是传统学院派的主流数学家，但非常擅长从整数中发现有趣规律。后来人们熟知的两个代表性结果，就是三位数的 **495** 和四位数的 **6174**。

他发现了一个有趣现象：

对一个三位数反复做同一个操作，很多时候最后都会得到 **495**，并且停在这里不再变化。

这个 495 就被称为三位数的“数字黑洞”（也常叫 Kaprekar 常数）。

你可能会好奇：

1. 这个过程到底怎么做？
2. 为什么最后常常会到 495？
3. 用 Python 能不能把每一步都打印出来？

这就是一个典型的 **数学规律 + 编程验证** 问题。

## 二、对应的数学知识

### 1. 三位数字黑洞的操作规则

设当前数字是一个三位数（不足三位要补齐0,比如**17** 会补成 **017**）。

每一步都做：

1. 把这个数的 3 个数字从大到小排成一个数 `A`
2. 再把同样 3 个数字从小到大排成一个数 `B`
3. 计算 `A - B`，得到下一步数字

重复这个过程。

例如从 352 开始：

- 第1步：532 - 235 = 297
- 第2步：972 - 279 = 693
- 第3步：963 - 369 = 594
- 第4步：954 - 459 = 495
- 第5步：954 - 459 = 495（不再变化）

### 2. 为什么说它是“黑洞”

因为很多不同的起点，都会被“吸”到同一个终点 495。

当到达 495 后：

$$954 - 459 = 495$$

结果等于自己，所以会一直停在 495。

### 3. 特殊情况说明

如果三位数的三个数字都相同（例如 111、222、999）：

$$111 - 111 = 0$$

会进入 000，不会到 495。

所以教材中通常会加一个条件：

- 三位数中至少有两位不同

### 4. 数字黑洞背后的数学道理

三位数做“降序减升序”时，本质上是把同一组数字重新排列后相减。

设三位数字按大小关系写成 $x \ge y \ge z$，那么：

- 降序数是 $100x + 10y + z$
- 升序数是 $100z + 10y + x$

两者相减：

$$
(100x + 10y + z) - (100z + 10y + x) = 99(x-z)
$$

这说明每一步结果一定是 99 的倍数。又因为三位数范围有限，过程不可能无限产生新值，最终会进入稳定状态或循环。对“三位且不全相同”的情况，这个稳定点就是 495：

$$954 - 459 = 495$$

所以 495 会像“吸引点”一样，把很多起始数字吸过去，这就是数字黑洞现象的数学原因。

## 三、数学建模

### 问题分析

1. 输入：一个三位整数 `n`（三个数字不全相同）
2. 输出：
   - 每一次运算过程（最大数 - 最小数 = 新数字）
   - 直到结果变成 495

### 模型步骤

1. **拆分数字**
   - 百位：`a = n // 100`
   - 十位：`b = n // 10 % 10`
   - 个位：`c = n % 10`

2. **构造最大数与最小数**
    - 最大数：百位取三个数字中的最大值，个位取最小值，中间位取排序后的中间值
    - 最小数：百位取最小值，个位取最大值，中间位同样取排序后的中间值
    - Python 常用函数说明：
       - `max(a, b, c)`：返回 3 个数字中的最大值
       - `min(a, b, c)`：返回 3 个数字中的最小值
       - `sorted([a, b, c])`：把 3 个数字按从小到大排序，结果是一个列表，例如 `sorted([3, 5, 2])` 得到 `[2, 3, 5]`，其中中间值可用 `sorted([a, b, c])[1]` 取得

3. **迭代计算**
   - 计算：`n = 最大数 - 最小数`
   - 记录本次过程
   - 用结果继续下一轮

4. **停止条件**
   - 若结果是 495，停止（到达数字黑洞）

## 四、Python 代码实现

下面代码尽量使用初学者容易理解的写法：

```python
# 说明主题
print("# 卡普雷卡数字黑洞 三位数 495")
# 读入一个三位数（默认满足：三个数字不全相同）
n = int(input("请输入一个三位数（数字不全相同）："))

# 不断迭代，直到进入 495
while n != 495:
    # 拆分数字
    a = n // 100 # 百位
    b = n // 10 % 10 # 十位
    c = n % 10 # 个位

   # 组成最大数和最小数（降序数 - 升序数）
    max_num = max(a, b, c) * 100 + sorted([a, b, c])[1] * 10 + min(a, b, c)
    min_num = min(a, b, c) * 100 + sorted([a, b, c])[1] * 10 + max(a, b, c)

   # 得到下一轮数字，并打印本轮过程
    n = max_num - min_num
    print(f"{max_num} - {min_num} = {n}")

# 到达数字黑洞
print("进入数字黑洞：495")
```
