# 数字黑洞：三位数为什么总会走向 495

## 一、故事内容

有一位印度数学家叫 **D. R. Kaprekar（卡普雷卡尔）**，他非常喜欢研究“看起来普通、但反复操作后会出现固定结果”的数字游戏。

他发现了一个有趣现象：

对一个三位数反复做同一个操作，很多时候最后都会得到 **495**，并且停在这里不再变化。

这个 495 就被称为三位数的“数字黑洞”（也常叫 Kaprekar 常数）。

你可能会好奇：

1. 这个过程到底怎么做？
2. 为什么最后常常会到 495？
3. 用 Python 能不能把每一步都打印出来？

这就是一个典型的 **数学规律 + 编程验证** 问题。

### 2. 三位数字黑洞的操作规则

设当前数字是一个三位数（不足三位要补前导 0）。

每一步都做：

1. 把这个数的 3 个数字从大到小排成一个数 `A`
2. 再把同样 3 个数字从小到大排成一个数 `B`
3. 计算 `A - B`，得到下一步数字

重复这个过程。

例如从 352 开始：

- 第1步：532 - 235 = 297
- 第2步：972 - 279 = 693
- 第3步：963 - 369 = 594
- 第4步：954 - 459 = 495
- 第5步：954 - 459 = 495（不再变化）

### 3. 为什么说它是“黑洞”

因为很多不同的起点，都会被“吸”到同一个终点 495。

当到达 495 后：

$$954 - 459 = 495$$

结果等于自己，所以会一直停在 495。

### 4. 特殊情况说明

如果三位数的三个数字都相同（例如 111、222、999）：

$$111 - 111 = 0$$

会进入 000，不会到 495。

所以教材中通常会加一个条件：

- 三位数中至少有两位不同

### 5. 数字黑洞背后的数学道理

三位数做“降序减升序”时，本质上是把同一组数字重新排列后相减。

设三位数字按大小关系写成 $x \ge y \ge z$，那么：

- 降序数是 $100x + 10y + z$
- 升序数是 $100z + 10y + x$

两者相减：

$$
(100x + 10y + z) - (100z + 10y + x) = 99(x-z)
$$

这说明每一步结果一定是 99 的倍数。又因为三位数范围有限，过程不可能无限产生新值，最终会进入稳定状态或循环。对“三位且不全相同”的情况，这个稳定点就是 495：

$$954 - 459 = 495$$

所以 495 会像“吸引点”一样，把很多起始数字吸过去，这就是数字黑洞现象的数学原因。

## 三、数学建模

### 问题分析

1. 输入：一个三位整数 `n`（三个数字不全相同）
2. 输出：
   - 每一次运算过程（最大数 - 最小数 = 新数字）
   - 直到结果变成 495

### 模型步骤

1. **拆分数字**
   - 百位：`a = n // 100`
   - 十位：`b = n // 10 % 10`
   - 个位：`c = n % 10`

2. **构造最大数与最小数**
    - 最大数：百位取三个数字中的最大值，个位取最小值，中间位取排序后的中间值
    - 最小数：百位取最小值，个位取最大值，中间位同样取排序后的中间值

3. **迭代计算**
   - 计算：`n = 最大数 - 最小数`
   - 记录本次过程
   - 用结果继续下一轮

4. **停止条件**
   - 若结果是 495，停止（到达数字黑洞）

## 四、Python 代码实现

下面代码尽量使用初学者容易理解的写法：

```python
print("# 卡普雷卡数字黑洞 三位数 495")
n = int(input("请输入一个三位数（数字不全相同）："))

while n != 495:
    # 拆分数字
    a = n // 100
    b = n // 10 % 10
    c = n % 10

    # 组成最大数和最小数
    max_num = max(a, b, c) * 100 + sorted([a, b, c])[1] * 10 + min(a, b, c)
    min_num = min(a, b, c) * 100 + sorted([a, b, c])[1] * 10 + max(a, b, c)

    n = max_num - min_num
    print(f"{max_num} - {min_num} = {n}")

print("进入数字黑洞：495")
```

## 六、总结

1. **规律本质**：同一组数字做“降序减升序”，会不断把数字推向稳定状态。
2. **三位黑洞**：大多数三位数（至少两位不同）最终会到达 495。
3. **特殊输入**：像 111、222 这种三个数字相同的数，会走到 000。
4. **编程价值**：用 Python 把每一步打印出来，比只看公式更容易理解数字黑洞的吸引过程。

这个例子说明：**数学发现规律，编程验证规律**。两者结合，学习效果最好。
